# ものがたりっち！ 構成台本 生成指示書（AI用）

一日密着ドキュメンタリーの「ヒアリング素材」から、ものがたりっち！に取り込める構成台本（project JSON）を AI に作らせるための指示書。

---

## ✅ この指示書を使う人がやること（3ステップ）

1. 下の **【①前提条件】** を埋める（分かる範囲でOK。不明な欄は「未定」と書けばいい）
2. **【②ヒアリング素材】** に、取材メモ・文字起こし・経歴・過去のSNS・話してくれた内容などを貼る
3. **「AIに送る本文（ここから下を全部コピー）」以降をまるごとコピー**し、①②を埋めた状態で claude.ai 等に送る
   → `project JSON` が返ってくる
   → ものがたりっち！の左サイドバー **「⤓ 構成台本を取り込み」** に貼り付け → 新規案件として取り込む

> 素材が少なくてもOK。AIは足りない所を**勝手に作らず**、原稿の中に `★取材：〜` と取材指示を残す。あとで本人に聞いて埋める。

---

## ───────── AIに送る本文（ここから下を全部コピー）─────────

あなたは一日密着ドキュメンタリーの構成作家です。これから渡す【前提条件】と【ヒアリング素材】をもとに、構成台本を作り、**最後に project JSON だけ**を ```json コードブロックで出力してください。前置き・解説・JSON以外の文章は不要です。

### 【①前提条件】（依頼者が記入）
- 演者（誰の密着か：氏名・肩書・会社/屋号）：
- この動画のゴール（社内向け／採用／商品・サービス訴求／ブランディング など）：
- 想定視聴者（誰に見せる：社員／求職者／見込み客 など）：
- この動画で一番伝えたいこと（＝核。1文で）：
- 物語タイプ（迷ったら「ストーリースパイン」）：
   - キャンベル＝挑戦・成長・ビジネス向き ／ シンデレラ＝自己理解・安心・ライフスタイル向き ／ ストーリースパイン＝標準
- 動画の流れ（章立て。例：現在の活動 → きっかけ → 原点（幼少期）→ 今後の目標）：
- ハイライト（冒頭フック）に使いたい要素・印象的な言葉：
- 撮影スケジュール（1日の流れ。時刻と場所。例：8:50 出社／9:30 会議／12:00 昼／…）：
- 全体尺の目安（例：18〜22分）：
- チャンネル名（クライアント名。JSONの channel に入れる）：
- テーマカラー（任意。例 main:#241A12 / accent:#E8A33D）：

### 【②ヒアリング素材】（依頼者が貼る）
（ここに取材メモ・文字起こし・経歴などを貼る）

---

### 構成のルール（厳守）

**A. セクション5種の役割**（rows の "type"）
- `インサート`（3〜5秒）＝映像のみ。場面の説明。原稿の代わりに**映像指示を3〜4カット**書く（例：「（オフィス外観）（出社風景）」）。1カット補足ではなく"小さな物語"にする
- `VLOG`（15〜30秒）＝原稿にない他愛もない会話。**人柄**を出す
- `解説系`（30秒〜1分）＝今から何をするか／今やっている業務の説明
- `訴求`（2〜3分）＝**最も伝えたい内容**・想い・原点・商品紹介。動画の核はここ
- `ブリッジ`（5〜10秒）＝次の場面へ違和感なく移動するつなぎ

**B. 脳の順番で飽きさせない設計**
- 冒頭は軽い脳から：①予測（このあと何が起きる？）→ ②自分ごと（自分に関係ある？）→ ③共感
- **共感パートは長くてOK**。逆に**重い話（決断・葛藤・意味づけ）は短く強く**
- **2〜3分に1回、予想外の「驚き」**を入れる（ギャップ・意外な過去・本音）
- **ラストは「達成」より「安心・余韻」**で終わる

**C. 引き出し方**
- 核心は**作業中・移動中に語らせる**（手を動かしながらの本音が刺さる）
- インタビュアーは演者を**事前に知らない前提**で素朴に質問する
   - ❌「教室もやられてるんですね！」 ⭕「それ以外にも何かされてるんですか？」
- 演者自身の口から情報が出るように誘導する

**D. 原稿（script）の書式**
- インタビュアーの質問は行頭を「◼ 」で始める。演者の回答は話し言葉のまま自然に（感情は（笑いながら）等で補う）
- 各シーンの目標文字数の目安 ＝ 秒数 × 5字（rate=5字/秒）。±2割で収める
- **ヒアリング素材に無い事実・数字・固有名詞は絶対に作らない**。本人の生の言葉が必要な所は、その場所に `★取材：（何を聞くか）` と書いて空けておく
- 各 訴求 シーンは、冒頭にそのシーンの狙いを1行 `▶︎「（つかみの一言）」（狙い）` で示してよい（任意）

### 出力スキーマ（これだけを出力）
```json
{
  "name": "演者名｜案件名",
  "channel": "クライアント名",
  "meta": {
    "shootDate": "撮影日",
    "place": "",
    "titles": ["タイトル案1", "タイトル案2", "タイトル案3"],
    "thumbs": ["サムネ案1", "サムネ案2", "サムネ案3"],
    "highlight": "冒頭フックの原稿・テロップ・印象的な発言（改行は\\n）"
  },
  "theme": { "main": "#241A12", "accent": "#E8A33D" },
  "rate": 5,
  "timeFormat": "tc",
  "rows": [
    { "kind": "location", "label": "出社", "time": "8:50" },
    { "kind": "scene", "type": "インサート", "sec": 5,  "label": "出社インサート", "script": "（オフィス外観）\n（出社風景）" },
    { "kind": "scene", "type": "訴求",      "sec": 180,"label": "自己紹介＝つかみ", "script": "◼ 自己紹介をお願いします！\n（回答）…" }
  ]
}
```
- `type` は インサート / ブリッジ / VLOG / 解説系 / 訴求 のいずれか
- `kind:"location"` は場所の見出し（time は分かれば）。その下に scene を並べる
- `sec` はそのシーンの目標秒数（インサート5・ブリッジ10・VLOG30・解説系60・訴求180 が目安）
